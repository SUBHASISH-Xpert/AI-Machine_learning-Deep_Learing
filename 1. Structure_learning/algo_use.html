Supervised · Regression

01
Linear Regression Predicts a continuous numeric output
Supervised
⚡ Use When You need to predict a numeric value (price, temperature, sales) and the relationship between input and output is roughly linear. Best when data is clean, features are numeric, and you need an interpretable model quickly.

Step-by-Step Process

1
Collect & Explore DataGather dataset with input features (X) and a numeric target (y). Check for nulls, outliers, and distribution.
2
Feature Selection & EngineeringChoose relevant features. Normalize/scale values. Remove highly correlated redundant features.
3
Split DataDivide into training set (80%) and test set (20%) to evaluate generalization.
4
Fit the ModelModel learns the best-fit line: y = mx + b by minimizing Mean Squared Error (MSE) using Ordinary Least Squares or Gradient Descent.
5
EvaluateUse R² score, MAE, and RMSE to measure accuracy. Check residual plots.
6
PredictFeed new input features → get a continuous numeric prediction.

Real-World Use Cases

🏠 House price prediction 📈 Stock price forecasting 🌡️ Weather temperature 💰 Salary estimation ⚡ Energy consumption

Supervised · Classification

02
Logistic Regression Classifies into two or more categories
Classification
⚡ Use When You want to predict a category/class (Yes/No, Spam/Not Spam). Great as a baseline classifier when data is linearly separable and you need probability scores alongside predictions.

Step-by-Step Process

1
Prepare DataLabel encode target classes. Handle class imbalance (SMOTE or class weights).
2
Feature ScalingStandardize features (mean=0, std=1) — logistic regression is sensitive to scale.
3
Train the ModelSigmoid function maps linear output to probability (0–1). Uses Maximum Likelihood Estimation to find optimal weights.
4
Set Decision ThresholdDefault is 0.5: P ≥ 0.5 → Class 1, else Class 0. Tune threshold for precision/recall tradeoff.
5
EvaluateUse Accuracy, Precision, Recall, F1 Score, ROC-AUC curve.

Real-World Use Cases

📧 Spam detection 🏥 Disease diagnosis (cancer yes/no) 💳 Fraud detection 🎓 Pass/Fail prediction 📊 Customer churn

Supervised · Classification & Regression

03
Decision Tree Splits data using if-then rules in a tree structure
Interpretable
⚡ Use When You need a visual, explainable model. Works with both numeric and categorical data, requires minimal preprocessing, and handles non-linear relationships naturally. Ideal for stakeholder-facing models.

Step-by-Step Process

1
Select Root FeatureAlgorithm tries every feature and threshold. Picks the one that gives the best split — maximizing Information Gain (for classification) or minimizing MSE (for regression).
2
Recursive SplittingEach branch repeats the same process — best split feature is chosen at every node until stop criteria is met.
3
Stop ConditionStop when: max depth reached, min samples in leaf, or no further info gain. This prevents overfitting.
4
Pruning (Optional)Post-pruning removes branches that offer little predictive power, improving generalization.
5
PredictNew data travels from root to leaf by following feature conditions. Leaf node gives the prediction.

Real-World Use Cases

🏦 Loan approval 🩺 Medical diagnosis 🛒 Customer segmentation ⚙️ Fault diagnosis 📋 HR decision making

Ensemble · Classification & Regression

04
Random Forest Combines many decision trees via bagging
Ensemble
⚡ Use When You need high accuracy and robustness on tabular data. Handles missing values, outliers, and high-dimensional data well. A go-to algorithm when a single decision tree overfits. Great for medium-to-large datasets.

Step-by-Step Process

1
Bootstrap SamplingRandomly sample the training data with replacement N times to create N different datasets (one per tree).
2
Build N Decision TreesTrain each tree on its bootstrapped dataset. At each split, only a random subset of features is considered (prevents trees being too similar).
3
Aggregate PredictionsClassification → majority vote across all trees. Regression → average output of all trees.
4
Feature ImportanceMeasure how much each feature reduces impurity across all trees — great for feature selection.
5
Tune HyperparametersKey params: n_estimators (more = better but slower), max_depth, min_samples_split.

Real-World Use Cases

💊 Drug discovery 🏦 Credit scoring 🌾 Crop yield prediction 🔐 Intrusion detection 🛍️ Recommendation systems

Supervised · Classification & Regression

05
Support Vector Machine (SVM) Finds the optimal hyperplane that separates classes
High-Dimensional
⚡ Use When Data has many features relative to samples (e.g. text classification, image data). Excellent when classes are clearly separable. Works well for small-to-medium high-dimensional datasets. Not ideal for very large datasets (slow training).

Step-by-Step Process

1
Feature ScalingSVM is sensitive to scale. Standardize all features to the same range first.
2
Choose KernelLinear kernel for linearly separable data. RBF (Radial Basis Function) for non-linear data. Polynomial for curved boundaries.
3
Find Maximum-Margin HyperplaneSVM finds the decision boundary that maximizes the margin (gap) between the two classes. Points on the margin are "support vectors."
4
Soft Margin (C parameter)C controls the tradeoff: high C → fewer misclassifications but narrow margin. Low C → wider margin but allows some misclassification.
5
Evaluate & PredictWhich side of the hyperplane a new point falls on determines its class.

Real-World Use Cases

📝 Text classification (NLP) 🖼️ Image recognition 🧬 Bioinformatics 🤚 Handwriting recognition 📡 Face detection

Supervised · Instance-Based

06
K-Nearest Neighbors (KNN) Classifies based on the K closest data points
Lazy Learner
⚡ Use When Dataset is small to medium, patterns are local, and you want a simple non-parametric approach. No training phase — all computation is at prediction time. Best when feature space is low-dimensional.

Step-by-Step Process

1
Choose K valueK = number of neighbors to consider. Odd K avoids ties. Use cross-validation to pick the best K.
2
Normalize FeaturesKNN uses distance (Euclidean, Manhattan). Features must be on the same scale or one dominates.
3
Store All Training DataKNN is a "lazy learner" — no model is built. All training data is stored and used at prediction time.
4
Compute DistancesFor a new data point, calculate its distance to every training point.
5
Vote & PredictPick K nearest neighbors. Classification → majority class wins. Regression → average their values.

Real-World Use Cases

🎬 Movie recommendation 🛒 Product recommendation 🩺 Medical pattern matching 🖼️ Image similarity search 📍 Location-based services

Unsupervised · Clustering

07
K-Means Clustering Groups data into K natural clusters without labels
Unsupervised
⚡ Use When You have unlabeled data and want to discover hidden groupings or patterns. Useful for customer segmentation, data compression, and anomaly detection. Works best when clusters are spherical and roughly equal-sized.

Step-by-Step Process

1
Choose K (number of clusters)Use the Elbow Method: plot WCSS vs K and pick the "elbow" where adding more clusters gives diminishing returns.
2
Initialize CentroidsRandomly place K centroids in the feature space. K-Means++ initialization improves convergence speed and quality.
3
Assign Points to Nearest CentroidEach data point is assigned to the cluster whose centroid is closest (by Euclidean distance).
4
Recalculate CentroidsFor each cluster, compute the mean of all assigned points — this becomes the new centroid.
5
Repeat Until ConvergenceSteps 3–4 repeat until centroids stop moving or a max iteration limit is reached.

Real-World Use Cases

👥 Customer segmentation 🗺️ Geo-location clustering 🖼️ Image compression 📰 Document grouping 🔎 Anomaly detection
